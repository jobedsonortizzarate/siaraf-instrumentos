## Description

## Installation

```bash
$ npm install
```

## Running the app

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Test

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Acerca de filtros

Para usar los filtros genéricos dentro de las APIS
se puede mandar un arreglo de objetos con el siguiente formato de la siguiente manera:
el objeto cuenta con los siguientes campos:

-"property" nombre de la propiedad por la que se desea filtrar

-"comparison" filtro de comparación, los validos son 'EQUAL', 'LESS_THAN', 'MORE_THAN' y 'NOT', en el caso de texto 'EQUAL' funciona como like

-"value" valor por el que se desea filtrar

ejemplo numérico
{"property":"cveEncomienda","comparison":"LESS_THAN","value":"131"}
el valor por default del campo ‘comparison’ es 'EQUAL' por lo que, si se omite dicho campo, el filtro funcionará

ejemplo de búsqueda por like
{"property":"nombre","value":"SA"}

en caso de hacer pruebas con Postman, se agrega un campo en el tab Params con la llave filters[0] y el objeto de filtro, el 0 pertenece al índice del arreglo que mandara, si se quiere filtrar por más de una propiedad, duplicar el campo y cambiar al siguiente índice consecutivo
